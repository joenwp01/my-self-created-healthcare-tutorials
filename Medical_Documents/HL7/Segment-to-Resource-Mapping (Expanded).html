Station 11 i

Segment-to-Resource Mapping

FHIR 1/3

Converting HL7 v2 messages to FHIR is not a one-to-one rename. The two standards are organized on different principles. v2 is a flat list of segments where context is positional — a PID identifies the patient, and every segment that follows in the message belongs to that patient because of where it sits. FHIR is a graph of resources, where each resource (Patient, Encounter, Observation) stands alone and is connected to others through explicit reference fields like subject and encounter.

1. Setting the table

You already work with two segments routinely: MSH, the Message Header (transport metadata — sender, receiver, message type), and PID, the Patient Identification segment (name, date of birth, sex, MRN). The other segments below are the ones you will see most often when v2 messages carry clinical content. Each one becomes a different FHIR resource on the other side of the conversion.

Abbreviations used in this tutorial

HL7 v2
Health Level Seven, version 2 — the legacy pipe-delimited messaging standard most healthcare systems still emit today.
FHIR
Fast Healthcare Interoperability Resources — the modern JSON/XML-based standard. Pronounced "fire."
MSH
Message Header segment — transport envelope.
PID
Patient Identification segment — demographics and identifiers.
PV1
Patient Visit segment — visit class, location, attending provider, dates.
OBR
Observation Request segment — the order or panel a result was produced for.
OBX
Observation segment — one row per individual result, finding, or measurement.
DG1
Diagnosis segment — admit, working, or discharge diagnosis.
IN1
Insurance segment — coverage / payor information.
AL1
Allergy segment — patient allergy or adverse reaction.
ADT
Admission, Discharge, Transfer — the family of v2 messages that signal patient movement events.
ORU^R01
Observation Result, Unsolicited — the v2 message that carries lab/diagnostic results.
LOINC
Logical Observation Identifiers Names and Codes — terminology for lab tests, vital signs, and clinical observations.
ICD-10-CM
International Classification of Diseases, 10th Revision, Clinical Modification — the diagnosis code set.

From this point forward I will use the abbreviation only.

2. The basic mapping rule

At the simplest level, each v2 segment that carries clinical content corresponds to exactly one FHIR resource on the other side. Here is the canonical map for the six segments you will meet most often:

PID → Patient — demographics + identifiers PV1 → Encounter — a single visit or admission OBR → DiagnosticReport — grouped lab or imaging report OBX → Observation — a single result or measurement DG1 → Condition — an active or historical diagnosis IN1 → Coverage — an insurance plan record

That is the easy part. The rest of this tutorial is about the things this table does not show: how a single segment can map to different resources depending on what it carries, how the FHIR resources re-establish the relationships that v2 expressed by segment ordering, and how the converter generates the IDs that wire the resources together.
