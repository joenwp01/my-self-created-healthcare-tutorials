What is inside

Included assets

  • index.html — polished starter home page with links
  • demo.html — full interactive FHIR transformation viewer
  • docs/index.html — browser-friendly guide hub
  • docs/*.html — HTML versions of every guide
  • docs/*.md — original markdown source files
  • run_local_server.bat and run_local_server.sh — helper launchers
Best way to use it

Two simple options

  1. Fastest: extract the ZIP, open index.html, and bookmark it.
  2. Cleaner bookmark: run the included local server script, open http://localhost:8000, and bookmark that URL.
For long-term use, keep the extracted folder in a stable place such as Documents\FHIR-Demo. Moving the folder later can break a file:/// bookmark.

Demo flows

What the interactive demo covers

Patient + Encounter + Condition

See how registration and diagnosis data are separated, normalized, coded, linked into FHIR resources, and flattened for analytics.

MedicationRequest flow

Follow a prescribing event through parsing, RxNorm-style mapping, requester linkage, validation, and medication reporting output.

C-CDA or CSV into FHIR

Watch semi-structured XML-style or flat CSV input become extracted facts, reconciled coding, native FHIR, and warehouse rows.

FHIR API response into SQL warehouse row

Start with a FHIR API Bundle, extract nested fields, resolve references, and load a warehouse fact row.

Bookmarking guidance

Best browser method

Best overall: double-click run_local_server.bat on Windows or run bash run_local_server.sh on macOS/Linux, then visit http://localhost:8000 and bookmark that page. This feels most like a small local site.

Simplest: open index.html directly and bookmark it. This needs no terminal, but the bookmark depends on the folder staying in the same location.

Tip: once the home page is bookmarked, you can use its links to reach the demo and docs without bookmarking each file separately.
