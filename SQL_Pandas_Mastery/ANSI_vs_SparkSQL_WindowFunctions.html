Window Functions: ANSI SQL vs SparkSQL

ANSI SQL and SparkSQL share most syntax — but diverge in the corners that matter for analytical queries. Frame boundaries, NULL handling, and RANGE semantics all differ in subtle ways. This tutorial contrasts them side-by-side, then lets you practice rewriting ANSI queries in Spark dialect.

How This Tutorial Works

Each topic has two panels. The Demo panel shows the same problem solved in both dialects with live results and an explanation of differences. The Practice panel beneath it shows a new problem pre-solved in ANSI SQL, then asks you to fill in the SparkSQL equivalent. Per-blank ◔ reveal buttons help when you're stuck; the Submit button scores your attempt against the full solution.

Note on the engine: Both sides run through SQLite in the browser. SQLite follows most ANSI SQL window semantics faithfully, but lacks IGNORE NULLS, strict RANGE BETWEEN n PRECEDING, and some advanced frame expressions. Where behavior deviates from the ANSI standard, a note appears beneath the code.
