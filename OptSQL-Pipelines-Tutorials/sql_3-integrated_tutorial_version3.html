Quick Recap

Three Techniques, One Query

Before diving into exercises, here's a reminder of what each technique contributes and why combining them is so powerful.

CTEs (WITH … AS) — Break complex logic into named, readable steps. Each CTE is a temporary named result set that later CTEs and the main query can reference.
Window Functions (fn OVER(…)) — Compute rankings, running totals, and row-to-row comparisons without collapsing rows. They run after WHERE / GROUP BY, so wrapping them in a CTE lets you filter on their results.
Parameterized Queries (%s / :name) — Replace hard-coded values with placeholders so the database binds user input safely. This prevents SQL injection and makes queries reusable.

Here's how they fit together in a single statement:

Combined Pattern -- The SQL string (with CTE + window + placeholders) query = """ WITH filtered AS ( -- CTE: named step SELECT id, rep, region, amount FROM sales WHERE region = %s -- Parameter: safe binding ), ranked AS ( SELECT *, RANK() OVER ( -- Window: rank without collapse ORDER BY amount DESC ) AS sales_rank FROM filtered -- CTE referencing earlier CTE ) SELECT rep, amount, sales_rank FROM ranked WHERE sales_rank <= %s -- Parameter: top-N threshold """ -- Execute with safe binding cursor.execute(query, (region_input, top_n))

Why this works:

  • The first CTE filters by region using a parameter — no string concatenation.
  • The second CTE applies a window function on the already-filtered data.
  • The outer query filters on the window result using another parameter.
  • The entire SQL string is passed to cursor.execute() with a parameter tuple — fully injection-safe.
